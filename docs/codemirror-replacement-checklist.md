# CodeMirror Replacement Checklist

This branch treats CodeMirror as the editor engine, not as the owner of app UI.

## Implemented Editor Behavior

- Document body provider returns the latest editor text for save and preview transitions.
- Source position provider reports the current scroll position as a fractional source line position.
- Preview/Edit toggling preserves source position and restores editor readiness.
- Editor focus is cleared when edit mode is inactive and restored only when appropriate.
- Selection is remembered across preview/edit transitions.
- Document changes reset editor-local selection and undo history; undo must never bring back another document or empty the new document.
- Markdown editing uses a monospaced 14 pt font, readable line height, transparent background, and horizontal/vertical padding.
- Lines wrap to the editor width.
- The editor allows vertical scrolling only while edit mode is active.
- Image paste and image drag/drop insert markdown generated by the app image pipeline.
- Empty documents show a placeholder without inserting placeholder text into the document.
- Escape collapses an active selection and otherwise leaves editor focus alone.
- Command-E toggles preview.
- Command-Z / Shift-Command-Z, Command-X/C/V/A keep expected editor behavior.
- Command-F / Command-G / Shift-Command-G use app-owned search UI and commands, not CodeMirror's default web search panel.
- Find/replace supports match count, full-match highlighting, replace current, replace all, and a collapsed replace row.
- Spelling, grammar, smart quotes, smart dashes, and automatic text replacement should not alter Markdown source.
- The old `MarkdownTextView`/`NSTextView` editor implementation has been removed.

## CodeMirror-Specific Rules

- Do not call `evaluateJavaScript` on every keystroke.
- Do not let CodeMirror document replacement enter user undo history.
- Prefer resetting the CodeMirror editor state on document switches so per-document undo stacks cannot leak.
- Keep SwiftUI/AppKit as the owner of app UI such as find, toolbar actions, metadata panels, preview state, and image insertion.
- Keep the WebKit bridge narrow: document load, latest text sync, source position, focus/active state, shortcuts, and image insertion.

## Stabilized

- Document switch undo isolation.
- Preview/Edit round trip.
- Latest-text sync cost and save freshness.
- Native/custom find replacement for CodeMirror's default panel.
- Paste/drop parity through the shared pasteboard image pipeline.
- Long document scroll and overscroll behavior.

## Deferred By Choice

- Find close restoring editor focus/selection after search UI dismissal.
- Excluding editor padding from CodeMirror selection visuals.
- Further preview/edit scroll-sync tuning beyond the current fractional source-position behavior.
